# PRD: 포모온다

## 1. 한 줄 정의

감정적 투자 결정에 반복적으로 흔들리는 20~30대 개인투자자가 매수·매도 충동 순간의 판단 이유를 객관적으로 점검하도록, 최신 이슈와 판단 이유를 비교 분석하는 AI 메타인지 점검 카드를 제공하는 웹 서비스

## 2. 문제와 타깃

- 문제: 개인투자자는 급등락·뉴스·커뮤니티·수익 인증에 반복적으로 흔들려 감정 기반 투자 실패를 반복하지만, 자신의 판단 이유와 객관적 이슈를 대조할 수 있는 장치가 없다. 실패 경험이 쌓일수록 금융 플랫폼 신뢰와 잔존율이 하락하는 리텐션 리스크로 이어진다.
- 타깃: 삼성전자를 보유 중이고, 오늘 급락 뉴스를 보고 "지금 팔아야 하나"를 반복적으로 검색하고 있는 28세 직장인 투자자

## 3. 핵심 시나리오

1. 진입 → 랜딩 페이지에서 "지금 흔들리고 있나요?" CTA 클릭
2. 입력 → 종목명 입력 + 현재 고민 상태(관심/보유) 선택 + 판단 이유 한 줄 자유 입력
3. AI 처리 → 판단 이유 텍스트 + 해당 종목 최신 뉴스 API 결과를 Claude API에 전달, 감정/외부 자극 여부 분석
4. 결과 → 점검 카드 출력 (판단 요약 / 근거로 보기 어려운 부분 / 최근 이슈 연결 포인트 / 확인 질문 3개)
5. 다음 행동 → 판단 이유 재작성 또는 이 기준 저장 선택

## 4. 기능 범위

- Must: 3개 이내
    - 종목 등록 + 판단 이유 입력
    - 뉴스 API 기반 최신 이슈 수집 + Claude API 점검 카드 생성
    - 판단 이유 재작성 또는 기준 저장
- Won't
    - 매수·매도 추천 또는 종목 추천 기능
    - 로그인·회원가입·마이페이지
    - 푸시 알림 (급등락 시 점검 제안)
    - 비슷한 흔들림 사례 (타인 데이터 집계)
    - 포트폴리오 연동·수익률 추적
    - 차트·시세 데이터 표시
    - 다국어 지원

## 5. 화면 구성

- 화면 1 · 랜딩: 서비스 한 줄 설명, "메타인지 가동하기" CTA 버튼
- 화면 2 · 종목 등록: 종목명 텍스트 입력, 상태 선택(관심/보유), 현재 고민 상태 선택(매수 고민/매도 고민), 다음 버튼
- 화면 3 · 판단 이유 입력: 자유 텍스트 입력창(최대 200자), 입력 예시 3개 노출, 점검 시작 버튼
- 화면 4 · 로딩: "판단 이유와 최신 이슈를 비교하고 있어요" 문구 + 스피너
- 화면 5 · 점검 카드: 판단 요약 텍스트, 근거로 보기 어려운 부분, 최근 이슈 연결 포인트, 확인 질문 3개, 하단 CTA 2개(판단 이유 다시 쓰기 / 이 기준 저장하기)
- 화면 6 · 판단 이유 재작성: 기존 입력 이유 표시, 재작성 텍스트 입력창, 저장 버튼
- 화면 7 · 기준 저장 완료: 저장된 기준 텍스트 확인, "다른 종목 점검하기" CTA

## 6. AI 동작 정의

- 입력: 사용자 판단 이유 텍스트(최대 200자) + 종목명 + 뉴스 API에서 수집한 최신 이슈 3~5개(제목+본문 요약)
- 출력: JSON 고정

```json
{
  "summary": "판단 요약 1~2문장",
  "weak_points": "근거로 보기 어려운 부분 1~2문장",
  "news_connection": "최근 이슈와 연결 포인트 1~2문장",
  "check_questions": ["질문1", "질문2", "질문3"]
}
```

- 프롬프트 방향: 투자 판단을 돕는 메타인지 코치 역할. 사용자의 판단을 비판하거나 감정을 자극하지 않고 중립적·질문형 톤 유지. 매수·매도 추천 절대 금지. "~하세요" 형태의 지시형 표현 금지. 사용자가 스스로 판단 이유를 다시 보게 만드는 것이 목적.
- 실패 처리: API 응답 오류 또는 파싱 실패 시 "지금 점검이 어려워요. 잠시 후 다시 시도해주세요" 문구 노출 + 재시도 버튼

## 7. 기술 스택

- 프레임워크: Next.js (App Router)
- 스타일: Tailwind CSS
- AI API: Anthropic Claude API (claude-sonnet-4-6)
- 뉴스 API: Naver 검색 API (종목명 기반 뉴스 수집)
- DB: Supabase (기준 저장 데이터 보관)
- 배포: Vercel

## 8. 데이터

저장 항목:

- 종목명
- 판단 이유 원문
- AI 점검 카드 결과 (JSON)
- 재작성 이유 텍스트 (있는 경우)
- 저장된 기준 텍스트 (있는 경우)
- 생성 시각

세션 기반으로 저장, 로그인 없이 로컬 세션 ID로 구분

## 9. 마일스톤과 완료 기준

- [x]  M1: 더미 데이터로 랜딩 → 종목 등록 → 판단 이유 입력 → 점검 카드 → 재작성/저장 전체 화면 흐름 동작
- [ ]  M2: Claude API + Naver 뉴스 API 연결, 실제 점검 카드 생성 동작
- [ ]  M3: Vercel 배포, Supabase 기준 저장 연결
- 완료 기준: 투자 고민이 있는 사용자가 설명 없이 종목과 판단 이유를 입력하고 점검 카드를 받아 재작성 또는 기준 저장까지 완료할 수 있는 상태

## 에이전트 작업 규칙

- 마일스톤 단위로만 작업하고, 시작 전에 계획을 먼저 보여줄 것
- Won't 목록의 기능은 제안도 구현도 하지 말 것
- 마일스톤 완료 시 위 체크박스를 갱신할 것
