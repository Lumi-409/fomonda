# PRD: 포모온다

## 1. 한 줄 정의

투자자의 매수·매도 판단을 대신하지 않고, 지금 이 판단이 뇌동매매·FOMO·불안에 휩쓸린 건 아닌지 스스로 점검하도록 돕는 메타인지 AI 서비스. 궁극적으로는 근거 기반 투자 경험을 반복하게 해 투자에 대한 긍정적 경험을 쌓게 하는 것이 목표.

## 2. 문제와 타깃

- 문제: 개인투자자는 급등락·뉴스·커뮤니티·수익 인증에 반복적으로 흔들려 뇌동매매·FOMO성 판단을 반복하지만, 자신의 판단이 근거에 기반한 것인지 감정에 휩쓸린 것인지 스스로 점검할 장치가 없다. 실패 경험이 쌓일수록 금융 플랫폼 신뢰와 잔존율이 하락하는 리텐션 리스크로 이어진다.
- 타깃: 삼성전자를 보유 중이고, 오늘 급락 뉴스를 보고 "지금 팔아야 하나"를 반복적으로 검색하고 있는 28세 직장인 투자자

## 3. 심리학적 근거 (UX Writing / 결과 화면 설계 기준)

서비스의 문구와 결과 화면 구조는 아래 네 가지 심리학 개념을 근거로 설계한다.

- **인지적 재평가 (Cognitive Reappraisal)**: 감정적 반응을 다른 관점에서 재해석하도록 돕는 기법. 결과 화면에서 사용자의 판단을 "틀렸다"고 지적하지 않고, "다르게 볼 수 있는 지점"으로 제시하는 톤에 반영한다.
- **표현적 글쓰기 (Expressive Writing, Pennebaker)**: 감정을 글로 직접 표현하면 정서적 부담이 줄고 사고가 정리된다는 연구. 판단 이유를 자유 텍스트로 직접 쓰게 하는 입력 화면 설계의 근거가 된다.
- **메타인지 훈련 (Metacognitive Awareness)**: 자신의 사고 과정을 스스로 관찰하고 점검하게 하는 훈련. 결과 화면의 "스스로 점검할 질문" 섹션 설계에 반영한다.
- **손실회피/처분효과 (Loss Aversion / Disposition Effect)**: 손실 확정을 피하려 손실 종목을 과도하게 오래 보유하고, 이익은 성급하게 실현하는 편향. 보유종목의 매도 판단을 점검할 때 결과 화면 문구와 질문 구성에 이 편향을 고려한다.

## 4. 핵심 시나리오

1. 진입 → 온보딩 화면에서 이 서비스가 매수·매도를 대신 정해주지 않는다는 것과 무엇을 도와줄 수 있는지 확인 후 시작
2. 종목 검색 → 지금 고민 중인 종목을 검색해서 선택
3. 보유 여부 선택 → 지금 이 종목을 보유 중인지(Y/N) 선택 → 보유종목/관심종목으로 분류
4. 판단 이유 입력 → 지금 이 판단을 내린 이유를 자유 텍스트로 직접 작성 (예시 버튼으로 이탈 방지)
5. 로딩 → "메타인지 가동 중" 문구와 함께 판단 이유 + 관련 뉴스 비교 처리
6. 결과 확인 → 점검 카드 확인, 결과는 자동으로 해당 종목의 판단 타임라인에 기록됨
7. 다음 행동 → 그 자리에서 판단 이유를 다시 써보거나(기준 재작성), 내 종목 리스트로 이동

## 5. 기능 범위

- Must
    - 종목 검색 + 선택 (하드코딩 리스트, 추후 KRX API로 교체 가능한 구조로 분리)
    - 보유 여부 선택(Y/N) → 관심종목/보유종목 분류
    - 판단 이유 입력 (자유 텍스트 + 예시 버튼)
    - 뉴스 API 기반 최신 이슈 수집 + LLM 점검 카드 생성
    - 점검 결과의 종목별 판단 타임라인 자동 기록 (별도 저장 버튼 없음)
    - 내 종목 리스트 (관심/보유 탭) + 종목별 판단 타임라인 조회
- Won't
    - 매수·매도 추천 또는 종목 추천 기능
    - 실시간 가격·시세 표시 (객관적 투자 판단이라는 서비스 가치를 훼손할 수 있어 제외)
    - 급등락 알림·푸시 알림 (오히려 FOMO를 자극하는 요소라 방향성과 맞지 않아 제외)
    - 목표가·손절가 도달 알림 (이번 스코프 제외, 추후 기능 후보로만 남겨둠)
    - 로그인·회원가입 (세션 기반 개인용 형태로 유지)
    - 비슷한 흔들림 사례 (타인 데이터 집계)
    - 포트폴리오 연동·수익률 추적
    - 차트 표시
    - 다국어 지원

## 6. 화면 구성

- 화면 1 · 온보딩: 이 서비스가 투자 판단을 대신해주지 않는다는 것을 명확히 알리고, 무엇을 도와줄 수 있고 무엇을 얻어갈 수 있는지 간단히 설명. "시작하기" CTA
- 화면 2 · 종목 검색: 매수/매도를 고민 중인 종목을 검색. 종목명+코드는 하드코딩된 리스트로 우선 구현하되, 추후 KRX 상장종목정보 API로 교체할 수 있도록 검색 로직을 분리된 모듈로 구현
- 화면 3 · 보유 여부 선택: 현재 보유 중인지(Y/N) 선택. Y면 보유종목, N이면 관심종목으로 분류
- 화면 4 · 판단 이유 입력: 자유 텍스트 입력창(최대 200자), 이탈 방지용 예시 버튼 노출, 점검 시작 버튼
- 화면 5 · 로딩: "메타인지 가동 중" 문구 + 스피너
- 화면 6 · 결과 화면: 판단 요약 / 약한 근거 / 관련 객관적 자료(뉴스 기반, Naver 뉴스 API 연동) / 스스로 점검할 질문 2~3개. 하단 버튼은 2개만 노출 — "기준 재작성"(primary, 강조) / "리스트 보러가기"(secondary). 화면 진입과 동시에 별도 저장 버튼 없이 판단 타임라인에 자동 기록
- 화면 7 · 내 종목 리스트: 관심/보유 탭으로 구분, 종목 클릭 시 해당 종목의 판단 타임라인(과거 판단 이유와 결과 이력) 확인 가능

## 7. AI 동작 정의

- 입력: 사용자 판단 이유 텍스트(최대 200자) + 종목명 + 뉴스 API에서 수집한 최신 이슈 3~5개(제목+본문 요약)
- 출력: JSON 고정

```json
{
  "summary": "판단 요약 1~2문장",
  "weak_points": "근거로 보기 어려운 부분 1~2문장",
  "news_connection": "관련 객관적 자료(뉴스) 연결 포인트 1~2문장",
  "check_questions": ["질문1", "질문2"]
}
```

- `check_questions`는 2~3개 사이로 가변
- 프롬프트 방향: 투자 판단을 돕는 메타인지 코치 역할. [[3. 심리학적 근거]]에 정리한 인지적 재평가·표현적 글쓰기·메타인지 훈련·손실회피/처분효과 관점을 반영해, 사용자의 판단을 비판하거나 감정을 자극하지 않고 중립적·질문형 톤 유지. 매수·매도 추천 절대 금지. "~하세요" 형태의 지시형 표현 금지. 사용자가 스스로 판단 이유를 다시 보게 만드는 것이 목적.
- 실패 처리: API 응답 오류 또는 파싱 실패 시 "지금 점검이 어려워요. 잠시 후 다시 시도해주세요" 문구 노출 + 재시도 버튼

## 8. 기술 스택

- 프레임워크: Next.js (App Router)
- 스타일: Tailwind CSS
- LLM API: OpenAI gpt-5.4-mini (Responses API + Structured Outputs로 JSON 스키마 고정), 서버 API route에서만 호출하고 API 키는 `OPENAI_API_KEY` 환경변수로 관리
- 뉴스 API: Naver 검색 API (종목명 기반 뉴스 수집)
- 종목 정보 API: 금융위원회 KRX 상장종목정보 & 주식시세정보 API (M1에서는 하드코딩 리스트 사용, 추후 이 API로 교체)
- DB: Supabase (종목별 판단 타임라인 보관)
- 배포: Vercel

## 9. 데이터

저장 항목:

- 종목 리스트: 종목명, 종목코드, 보유 여부(Y/N)
- 종목별 판단 타임라인 (종목당 여러 건 누적)
    - 판단 이유 원문
    - AI 점검 카드 결과 (JSON)
    - 생성 시각

세션 기반으로 저장, 로그인 없이 로컬 세션 ID로 구분. 별도의 "저장하기" 동작 없이 결과 확인 시점에 자동으로 타임라인에 기록된다.

## 10. 마일스톤과 완료 기준

- [x]  M1: 더미 데이터로 온보딩 → 종목 검색 → 보유 여부 선택 → 판단 이유 입력 → 로딩 → 결과(자동 타임라인 기록) → 내 종목 리스트/타임라인 조회까지 전체 화면 흐름 동작
- [x]  M2: OpenAI gpt-5.4-mini(Responses API + Structured Outputs) + Naver 뉴스 API 연결, 실제 점검 카드 생성 동작 확인. 종목 검색 KRX API 연동은 이번 스코프에서 제외(하드코딩 리스트 유지)
- [x]  M3: Vercel 배포(GitHub 연동 자동 배포) 및 Supabase에 종목별 판단 타임라인 영구 저장 연결 확인
- 완료 기준: 투자 고민이 있는 사용자가 설명 없이 종목을 검색하고 보유 여부와 판단 이유를 입력해 점검 카드를 받고, 그 결과가 자동으로 타임라인에 남아 내 종목 리스트에서 이력을 다시 확인할 수 있는 상태

## 에이전트 작업 규칙

- 마일스톤 단위로만 작업하고, 시작 전에 계획을 먼저 보여줄 것
- Won't 목록의 기능은 제안도 구현도 하지 말 것
- 마일스톤 완료 시 위 체크박스를 갱신할 것
- Naver 뉴스 API, KRX API, OpenAI API 등 외부 API 연동 작업을 시작하기 전에는 어떤 정보(키, 엔드포인트, 환경변수 등)가 필요한지 먼저 사용자에게 확인할 것
